Package

Let's have a look at the contents of the current package:

  • 01 - uAdmin Template (PHP)
    Includes the abstract PHP version. This version can come really handy as a starting point if you would like to build a custom PHP application. It can help you in many ways:
    • Templating
      Common code used only once and included in each page (files in /inc folder).
    • Main Navigation Auto Creation
      In the /inc/config.php file you can alter the $primary_nav[] array accordingly and the main navigation will be auto created for you on the fly.
    • Variables
      In the /inc/config.php file you can set some variables that used throughout uAdmin such as the layout mode (to fixed) or the active theme.
    • Demo Tasks
      Some code used for demonstration purposes, eg creating table rows with dummy content.
  • 02 - uAdmin Template (HTML)
    Includes the abstract HTML version. This version can be used with any server side language/framework you prefer or currently working with.
  • 03 - Extras
    Includes PSD files for Glyphicons Pro and uAdmin logo.
  • Documentation
    Of course, it includes current documentation!

Bootstrap

Bootstrap is one of the most popular front-end frameworks at the moment and is full of features. It is easy to use and it offers a large variety of CSS as well as Javascript components.

uAdmin was built on top of this framework and it extends it to a large degree. It has its own unique style and supports all of the features the Bootstrap framework has to offer. Attention to detail was given, so that every aspect of it to feel part of the theme and not just another feature.

If you are unfamiliar with the framework, its official website will help you out a lot but if you just want to start quickly, you can easily replicate any feature or component we already present in uAdmin. What is vital to know, is that the two files the framework consists of (required for uAdmin), are the following.

  • /css/bootstrap.css (unaltered, original framework)
  • /js/vendor/bootstrap.min.js (unaltered, original framework)

The above files include all the features the framework has, except for the default Glyphicons set. Those were not included because Glyphicons PRO ($59) and FontAwesome were used (superior, font-based icon solutions).

HTML

uAdmin layout is clean and simple as you can see at the following block of code. The available options, are a fixed header (top or bottom) and a fixed layout (in the PHP version these options can be set from the /inc/config.php file). 

<!-- Body -->
<!-- Add the class .fixed to <body> for a fixed layout on large resolutions (min: 1200px) -->
<!-- <body class="fixed"> -->
<body>
    <!-- Page Container -->
    <div id="page-container">
        <!-- Header -->
        <!-- Add the class .navbar-fixed-top or .navbar-fixed-bottom for a fixed header on top or bottom respectively -->
        <!-- <header class="navbar navbar-inverse navbar-fixed-top"> -->
        <!-- <header class="navbar navbar-inverse navbar-fixed-bottom"> -->
        <header class="navbar navbar-inverse">
            Logo, widgets etc..
        </header>
        <!-- END Header -->

        <!-- Inner Container -->
        <div id="inner-container">
            <!-- Sidebar -->
            <aside id="page-sidebar">
                Search and Main Navigation
            </aside>
            <!-- END Sidebar -->

            <!-- Page Content -->
            <div id="page-content">
                Main Content
            </div>
            <!-- END Page Content -->

            <!-- Footer -->
            <footer>
                Copyright etc..
            </footer>
            <!-- END Footer -->
        </div>
        <!-- END Inner Container -->
    </div>
    <!-- END Page Container -->
</body>
<!-- END Body -->

CSS

You can see below the CSS files along with their description.

  • /css/fonts/
    Folder that contains the fonts used in /css/plugins.css by Glyphicons PRO and FontAwesome
  • /css/themes/
    Folder that contains all the available themes. Just include the one of your choice after the main.css and before the themes.css. To create your own, simply copy-paste one of the existing, give it a new name and alter the colors to match your preference. If you would like the new color to be available for individual elements, add your new color theme to the themes.css file, the same way we did (check the file). If you are using the PHP version remember that you can set an active theme from the /inc/config.php file.
    • deepblue.css
    • deepgreen.css
    • deeppurple.css
    • deepwood.css
  • /css/bootstrap.css
    The Bootstrap framework in its original form, unaltered. It is a good practice not editing this file directly. It is better overwriting the classes you want (in the following files), so that a framework update is easier!
  • /css/plugins.css
    Styles used for the following plugins. Their original style is altered to match with uAdmin design.
    • FontAwesome
    • FullCalendar
    • Bootstrap Wysihtml5
    • Bootstrap DateRangePicker
    • Bootstrap Datepicker
    • Bootstrap Timepicker
    • Bootstrap Colorpicker
    • Dropzone.js
    • Chosen
    • JQVMap
    • Rainbow
    • Glyphicons PRO
    • GEMICON Icons
    • Select2
    • CSS3 ANIMATION CHEAT SHEET
  • /css/main.css
    This is the main stylesheet of uAdmin. The styles for the layout, uAdmin elements, as well as the overwrite and extension of Bootstrap styles exist here. Below, the sections of the file are presented.
    • LOGIN (Login page layout structure and elements)
    • MAIN LAYOUT (Main page layout structure)
    • HEADER WIDGETS (Widgets styles + extend of Bootstrap styles)
    • NAVIGATION (Search + Primary Nav + Nav info + Nav dash)
    • MAIN CONTENT (Template specific styles such as Dash Tiles)
    • BOOTSTRAP OVERWRITE + EXTEND STYLES (Overwrite and extend the default bootstrap styles + Datatables plugin)
    • HELPERS (Some helper classes)
    • GEMICON ICONS
    • RESPONSIVE
    • RETINA
  • /css/themes.css
    This is the themes stylesheet. You must include it last, after all other stylesheets. The classes included in this file can be used to alter the color, the background color or the border color of an element individually. It contains all available colors themes.

If you would like to edit a specific section of uAdmin, simply find the appropriate label in /css/plugins.css or /css/main.css files and change it to your own needs. Even better, we would suggest creating and including in the template a new CSS file named custom.css and apply your style overwrites there. This way, it will be easier for you to update the template's files when we release an updated version.

JavaScript

You can see below the Javascript files along with their description.

  • /js/ckeditor/*
    Required for using the CKEditor on a textarea. This folder contains all plugin's related files. You just have to include the ckeditor.js file in the page you would like to use it.
  • /js/helpers/excanvas.min.js
    Required for the Flot charts to work correctly in IE8 (loaded only in IE8)
  • /js/helpers/gmaps.min.js
    Required for Google Maps functionality, must be included along with the Google Maps API for using the maps
  • /js/helpers/jqvmap.maps.js
    JQVmaps data! Include it in your page if you want to use a vector map with the JQVmap plugin!
  • /js/vendor/bootstrap.min.js
    Javasript components of Bootstrap framework
  • /js/vendor/jquery-1.11.1.min.js
    The popular Javascript library that greatly reduces the amount of code you must write. This is the local file and is only included in case the version from google apis cannot be loaded for some reason (eg working local without a network connection available)
  • /js/vendor/modernizr-respond.min.js
    Modernizr, a browser feature detection library and
    Respond.js, enabling responsive CSS code on browsers that do not natively support it
  • /js/plugins.js
    All Javascript plugins minimized and included in a single file
  • /js/main.js
    Initialization of various plugins as well as Javascript code used in uAdmin features such as main navigation functionality

The Javascript code required and used throughout uAdmin, exists in /js/main.js. In some pages, because some functionality is needed individually, there is some initialization code at the bottom of each file only (as it happens in the index.php for example).

There are a lot of examples in uAdmin but for advanced usage of some plugins you can check out their websites where you could find more advanced examples. I've included their links in the last section of this documentation.

At last, all Javascript files are included at the bottom of the page (/inc/footer.php for the PHP version), except from the /js/vendor/modernizr-respond.min.js which needs to be loaded in the <head> tag (/inc/top.php for the PHP version).

Retina

uAdmin is retina ready! This means that it looks crystal clear on retina displays found on high-end mobile or other devices. If such display is available, the high resolution images will be served. If you add extra images and you want to have a retina ready interface, just add along with the original images (in the same folder), the same images with double pixels and append the text @2x to their original name. They will automatically be served if necessary! Just have a look at /img/template/ folder for an example. You only have to use the default image in your app. The high res image (@2x) will be served automatically if needed.

*only exception are the CSS Image Sprite based Icons
